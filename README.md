# 鉄輪式戦車レース 演出アプリ MVP

Discord画面共有を想定した、戦車レース1レース進行用のローカルWebアプリです。

## 起動方法

依存ライブラリをインストールしてから、Flaskアプリを起動します。

```bash
pip install flask
python app.py
```

起動後、ブラウザで以下を開きます。

- GM管理画面: `http://127.0.0.1:5000/control`
- 観客表示画面: `http://127.0.0.1:5000/display`

## 画面

### `/control`

GM用の管理画面です。

- ランク選択 D/C/B/A/S
- 番組選択 標準/高速/技巧/重装/射撃/荒れ場
- seed指定
- 1レース自動生成
- イベントの前後移動
- 現在イベント、出走表、イベント一覧の確認

### `/display`

プレイヤー・観客に見せる表示画面です。

- 三列エリアの現在配置
- 戦車カード
- 現在順位
- 実況文
- ラウンド名とseed

Discordで画面共有する場合は、GM管理画面ではなく `/display` を共有してください。

## Ver.0.4 追加仕様

`sim_engine.py` は現行Ver.0.5.1相当の簡易シミュレーターです。

Ver.0.4では、レース開始前に配置決めフェーズを行います。

- 前列狙い: `2d6 + 機動力 + 操縦性`、目標値9。
- 前列狙い成功: 前列配置、駆動力-1。
- 前列狙い失敗: 中列配置、駆動力-1、安定値-1。
- 出目2: 中列配置、駆動力-1、安定値-2。
- 出目11以上: 自動成功かつ大成功。前列配置、駆動力消費なし。
- 中列配置: 補正なし。
- 後列配置: 駆動力+1。

## 高速戦の構成変更

通常の高速戦は、Ver.0.4で以下の構成になりました。

- 逃げ2
- 追込2
- 万能2

旧来の逃げ3構成は、必要なら別番組として追加する想定です。

## Ver.0.5.1 AI・ルール候補

Ver.0.5.1では、番組ごとのAI補正を使わず、汎用AIと汎用ルールで番組差が出るようにしています。

- 行動AIはスコア制です。タイプ別の基本傾向に、序盤/中盤/終盤、現在エリア、暫定順位、HP、安定値、駆動力、弾薬を加味し、最終スコアによる重み付きランダムで行動を選びます。
- 弾薬0の射撃、駆動力不足の加速/危険な追い抜き、対象なしの接触など、意味のない行動は候補から除外されます。
- ターゲット選択もスコア制です。射撃は前列・上位・先行点持ち・安定値が低い相手を狙いやすく、接触は同エリア内の上位・低装甲・低安定・先行点持ちを狙いやすくなります。
- Aルール「移動予約中の隙」: 右側移動予約中に妨害を受け、制御判定が発生する場合、制御判定目標値+1。
- Bルール「重量衝突」: 接触妨害/危険接触が命中した場合、攻撃側装甲が防御側装甲を上回るほど制御判定目標値が上昇します。装甲差+1〜+2で+1、+3以上で+2。
- Cルール「防御走行の先行制限」: 防御走行では先行点を得られません。現在の実装では安定値+1のみです。
- 逃げ型は駆動力を一律-1しました。最低値は1です。
